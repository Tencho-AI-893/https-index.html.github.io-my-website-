# カルマ向け QRデジタル会員証（Google Workspaceのみ）

## 全体アーキテクチャ（1画面）

1. **Googleフォーム**で「ニックネーム」を登録
2. フォーム送信トリガー `onFormSubmit(e)` が実行され、`members` シートに会員情報を作成
   - `member_id` 自動発番（KM000001 形式）
   - 推測困難な `token` 生成
   - `card_url`（`.../exec?t=token`）生成
3. ユーザーは `card_url` を開くと、Webアプリ `doGet(e)` が `token` で会員を特定し会員証を表示
4. 会員証内の「チェックイン」ボタンで `doPost(e)` を呼び、`visits` シートに来店ログ追記
5. 管理者はスプレッドシートを見れば、**会員一覧（members）** と **来店ログ（visits）** を一元管理

---

## 1) スプレッドシート構成

### シート1: `members`
列定義（1行目ヘッダー固定）:

- `member_id`
- `nickname`
- `expires_at`
- `rank`
- `token`
- `card_url`
- `created_at`

### シート2: `visits`
列定義（1行目ヘッダー固定）:

- `timestamp`
- `member_id`
- `nickname`
- `rank`

> `setupSheets()` 実行で自動作成可能。

---

## 2) Googleフォーム構成（質問項目）

最低限の質問:

- **ニックネーム**（記述式・必須）

運用上の流れ:

1. フォーム回答先をこの運用スプレッドシートにリンク
2. Apps Script プロジェクトをこのスプレッドシートに紐づける（コンテナバインド）
3. フォーム送信時トリガーで `onFormSubmit` を実行
4. 送信のたびに `members` に新規会員行が追加される

---

## 3) セットアップ手順（クリック手順）

1. **Googleフォーム作成**
   - フォームを新規作成
   - 質問「ニックネーム」を追加（必須）
2. **回答先スプレッドシートを作成**
   - フォームの「回答」タブ → スプレッドシートアイコン → 新規作成
3. **Apps Scriptを開く**
   - スプレッドシート上部メニュー「拡張機能」→「Apps Script」
4. **ファイル作成**
   - `Code.gs` に `qr-membership/Code.gs` の内容を貼り付け
   - `Card.html` を新規作成し `qr-membership/Card.html` を貼り付け
5. **初期化関数実行**
   - Apps Script 上で `setupSheets` を選択して実行（初回のみ）
   - 権限承認を許可
6. **ウェブアプリとしてデプロイ**
   - 右上「デプロイ」→「新しいデプロイ」
   - 種類: 「ウェブアプリ」
   - 実行ユーザー: 自分
   - アクセス権: 「全員」
   - デプロイしてURLをコピー
7. **`WEB_APP_URL` を差し替え**
   - `Code.gs` の `CONFIG.WEB_APP_URL` にデプロイURLを貼る
   - 再デプロイ（バージョン更新）
8. **フォーム送信トリガー設定**
   - Apps Script 左メニュー「トリガー」→「トリガーを追加」
   - 実行関数: `onFormSubmit`
   - イベントソース: スプレッドシートから
   - イベント種類: フォーム送信時
9. **本番運用開始**
   - フォームで登録 → `members` に会員追加 → `card_url` を会員に案内

---

## 4) デプロイ後の動作確認（テスト手順）

1. フォームに1件登録
2. スプレッドシート `members` に以下が入ることを確認
   - `member_id`（KM000001 形式）
   - `token`（長いランダム文字列）
   - `card_url`
3. `card_url` をスマホで開いて会員証表示を確認
4. 「チェックイン」ボタンを押す
5. `visits` シートに1行追加されることを確認

---

## 5) 運用メモ（よくある詰まりポイント）

- `WEB_APP_URL` 未設定のままだと `card_url` がダミー文字列になる
- 再デプロイしないと `WEB_APP_URL` 変更が反映されない
- トリガー未設定だとフォーム送信しても `members` に追加されない
- フォーム項目名は「ニックネーム」推奨（異なる場合もコードはフォールバックする）
- トークン漏洩時は `regenToken(member_id)` を実行してURLを再発行する
